# Mock CI/CD Pipeline with Bash

This project is a mock CI/CD pipeline built to practice and solidify Bash scripting skills in a DevOps-style workflow.

The goal of this project is to simulate how code changes move from development to testing and then deployment using shell scripts, Docker, automated tests, and logging.

## Project Overview

The project simulates a basic CI/CD workflow:

1. A developer makes changes to the application code.
2. A pull request is created.
3. The CI pipeline runs automated checks such as testing and linting.
4. After the pull request is reviewed and approved, the CD pipeline runs.
5. The CD pipeline builds a new Docker image.
6. A temporary container is started to verify the deployment works.
7. The old running container is replaced with a new one.
8. The deployed application is tested again to confirm everything is working.

This project is not connected to a real Git hosting platform yet. Instead, the CI and CD steps are triggered manually using Bash scripts.

## Main Components
`app/`
Contains the mock Python application that is tested and deployed by the pipeline.

`tests/`
Contains automated tests for the application.
- `test_inprocess_api.py`
    Tests the application directly using `TestClient`.
- `test_deployment_api.py`
    Tests the deployed application using HTTP requests.

`scripts/ci.sh`
Simulates the Continuous Integration pipeline.

This script is responsible for running checks such as:
- automated tests
- linting
- basic validation before deployment

`scripts/cd.sh`
Simulates the Continuous Deployment pipeline.

This script is responsible for:
- building a Docker image
- starting a temporary test container
- checking that the deployment works
- stopping and removing the old container
- starting a new container from the latest image
- running deployment tests against the new container

`scripts/logging.sh`
A shared Bash logging library used by both the CI and CD scripts.

It helps keep logging consistent across the pipeline.

`logs/`
Stores output from the CI and CD scripts so pipeline activity can be reviewed later.

## How to Run
Make sure the scripts are executable:

```bash
chmod +x scripts/*.sh
```

Run the CI pipeline:

```bash
./scripts/ci.sh
```
Run the CD pipeline:

```bash
./scripts/cd.sh
```

## Requirements

This project expects the following tools to be installed:

- Bash
- curl
- Python
- Docker
- [`uv`](https://docs.astral.sh/uv/) package manager

This project uses `uv` to manage Python dependencies.

To install `uv`, run:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

After installing `uv`, sync the project dependencies:

```bash
uv sync
```
